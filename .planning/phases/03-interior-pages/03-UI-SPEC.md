## Spacing Scale

| Token | Value | Usage |

Exceptions: Touch targets for interactive elements (hamburger, links) minimum 44×44px per NAV-02.

## Typography

Exactly 4 rows. H3 card titles use Body size (16px) with font-bold weight — see note below.

| Role | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------|--------|-------------|----------------|-------|
| Label / Eyebrow | 12px (`text-xs`) | 400 (`font-normal`) | default | `tracking-widest` + uppercase | Section eyebrows, badge text, status tags — visually distinct via spacing + case, not weight |
| Body | 16px (`text-base`) | 400 (`font-normal`) | 1.6 (`leading-relaxed`) | default | Descriptive paragraphs, card body, capability bullets |
| Heading (H2) | 30–36px (`text-3xl` / `md:text-4xl`) | 700 (`font-bold`) | 1.25 (`leading-tight` / `leading-snug`) | `-0.02em` (`tracking-tight`) | Page section headings (H2 only) |
| Display (H1) | 48–72px (`text-5xl` / `md:text-6xl` / `lg:text-7xl`) | 700 (`font-bold`) | 1 (`leading-none`) | `-0.04em` (`tracking-tighter`) | Page hero headings only (solutions index, about, work) |

**H3 treatment note:** H3-level elements (BeliefGrid titles, card titles) use `text-base font-bold` (16px / weight 700). Differentiation from body text is by weight only, not size. This caps the effective type scale at 4 sizes.

Notes:
- Interior page H1 heroes use display scale: `text-5xl md:text-6xl` with `tracking-tighter leading-none` — one size down from home hero (`lg:text-7xl`) since these are sub-pages.
- "Who this is for" callout uses body size (16px) at `text-text-secondary` with a left `border-l-2 border-cyber-jade` treatment.
- BeliefGrid belief titles use `text-base font-bold` (H3-equivalent) with body paragraph below.
- Problem statement on solution detail uses body (16px) in a visually distinct block — `text-text-secondary` paragraph before the capabilities list, no special size increase.

Source: `DESIGN.md` typography table, `components/home/HeroSection.tsx`, `components/home/ApproachSection.tsx`

## Color

| Role | Hex | Tailwind Token | Usage |
|------|-----|----------------|-------|
| Dominant (60%) | `#0a0a0a` | `bg-background` | Page background for all interior pages |
| Secondary (30%) | `#111111` / `#1a1a1a` | `bg-surface` / `bg-surface-2` | Card backgrounds, section alternation |
| Accent (10%) | `#008684` | `cyber-jade` | Structural color — see reserved list below |
| Destructive | n/a | — | No destructive actions in this phase |

**Accent (`cyber-jade` / `#008684`) reserved for:**
1. Card hover border: `hover:border-cyber-jade` on solution and work cards
2. Inline text links: `text-cyber-jade hover:text-jade-strong` (e.g., "See our work →" pattern)
3. About page contact CTA button: `border border-cyber-jade text-cyber-jade` outline variant (matches FooterCTA.tsx pattern)
4. "Who this is for" callout left border accent: `border-l-2 border-cyber-jade`
5. Focus rings: `focus-visible:outline-cyber-jade` on all interactive elements
6. Numbered step accent in approach-style blocks: `text-cyber-jade/20`

**Volt Green (`#96D02C` / `volt-green`) rule — ONE per page:**
- Solutions index page: Volt Green CTA is the "Get in touch" / email link (hero-level CTA)
- Solution detail pages: NO Volt Green. Detail page CTA is cyber-jade outline button (Volt Green already present elsewhere in the viewport via navbar context — if no other volt-green is visible, one Volt Green link-style CTA at bottom of detail is acceptable, but default to cyber-jade outline)
- Work page: NO Volt Green. Use cyber-jade outline mailto button
- About page: NO Volt Green. Contact CTA uses cyber-jade outline button per ABOUT-02

**Status badges:**
- "Live": `bg-ev-teal/10 text-ev-teal` — `#1A7080` (soft teal pill)
- "In Development": `bg-amber-500/10 text-amber-400` — amber soft pill (no Tailwind token exists; use inline opacity variant)
- Grid Violet (`#7C3AED`): solution category tags on cards — `bg-grid-violet/10 text-grid-violet`

**Ghost / "Coming soon" card:**
- Border: `border border-dashed border-border-subtle` (`#2a2a2a`)
- Background: `bg-surface` (`#111111`)
- Text: `text-text-muted` (`#52525b`) centered

## Page-Level Layout Contracts

### Solutions Index (`/[locale]/solutions`)

**Primary focal point:** PageHero H1 + eyebrow. Secondary focal point: SolutionsGrid top-left card.

```
PageHero: eyebrow + "Solutions" H1 + 1-line sub
SolutionsGrid: 2-col md grid of 4 SolutionCards (each wrapped in <Link href="/solutions/{slug}">)
```

- Section background: `bg-background`
- Cards: existing `SolutionCard` pattern, full-card click target via wrapping `<Link>`
- No Volt Green CTA — solutions page drives to detail pages via card clicks
- Bottom: `FooterCTA` section (cyber-jade outline)

### Solution Detail (`/[locale]/solutions/[slug]`)

**Primary focal point:** SolutionDetailHero H1 + status badge. Secondary focal point: CapabilityList first item.

```
SolutionDetailHero: status badge + H1 (solution name) + problem statement paragraph
WhoThisIsFor: left-bordered callout with audience description
CapabilityList: numbered capabilities (min 3, max 6 per solution)
FooterCTA section OR inline cyber-jade outline mailto CTA at bottom
```

- Status badge positioned above H1, self-start aligned left
- Problem statement: `text-base font-normal text-text-secondary leading-relaxed` — full paragraph, NOT a bullet
- Capabilities come AFTER problem statement (SOL-04 requirement: problem-first)
- No Volt Green on detail pages

### Work Page (`/[locale]/work`)

**Primary focal point:** PageHero H1 + eyebrow. Secondary focal point: EV Fleet WorkCard (sole live project).

```
PageHero: eyebrow + "Our Work" H1 + 1-line sub
WorkCard: EV Fleet Operations (real card with tags + full description)
GhostCard × 2: "Coming soon" dashed cards
```

- Cards in a 3-col grid on lg, 2-col on md, 1-col on mobile
- "Live" status tag on EV Fleet card: `bg-ev-teal/10 text-ev-teal`
- Ghost cards: exact same dimensions as WorkCard, `border-dashed border-border-subtle`, text centered

### About Page (`/[locale]/about`)

**Primary focal point:** PageHero H1 belief statement. Secondary focal point: BeliefGrid first column heading.

```
PageHero: eyebrow + belief-statement H1 (NOT a founding story) + 1-line sub
BeliefGrid: 3 belief statements in columns
CurrentFocusParagraph: single paragraph about current focus, text-secondary
FooterCTA: cyber-jade outline mailto button (ABOUT-02)
```

- H1 on about is a belief/observation: e.g., "Infrastructure should outlast the problem it solves."
- No headshots (ABOUT-01)
- BeliefGrid: `md:grid-cols-3 gap-8`, each cell has a numbered accent (cyber-jade/20), `text-base font-bold` title, `text-base font-normal` body paragraph

## Interaction Contracts

### Card hover (all card types)

- Border transitions from `border-border-subtle` to `border-cyber-jade`
- Duration: `duration-150 ease-out`
- No transform, no shadow, no scale

### Link / text CTA hover

- Color transitions from `text-cyber-jade` to `text-jade-strong`
- Duration: `duration-150 ease-out`

### Button hover (cyber-jade outline)

- Background: `transparent` → `bg-cyber-jade/10`
- Duration: `duration-150 ease-out`
- No border change on hover (border stays cyber-jade)

### Focus rings (all interactive elements)

- `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-cyber-jade`
- No custom focus for mouse users (only keyboard-visible)

### Ghost card (no interaction)

- Static — no hover state. `cursor-default`.

### Status badge (no interaction)

- Static pill. No click, no tooltip.

### Language switcher (inherited from Phase 2)

- No changes needed. Uses existing Navbar component.

- Capabilities: outcome-based bullets, not feature lists. "Reduces idle time by routing against live traffic" not "Advanced routing algorithm."
