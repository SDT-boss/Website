  <action>
    Create a temporary SVG file at /tmp/logomark.svg that replicates the LogoMark.tsx geometry exactly: viewBox="0 0 36 36", background rect fill="#0a0a0a", cyber-jade ellipse (cx=24, cy=24, rx=6, ry=10, fill="#008684", transform="rotate(35, 24, 24)"), volt-green ellipse (cx=12, cy=12, rx=6, ry=10, fill="#96D02C", transform="rotate(35, 12, 12)"). <task type="auto">
  <name>Task 2: Derive apple-icon.png, favicon.ico, and og.png from icon.png</name>
  <files>
    app/apple-icon.png
    app/favicon.ico
    public/og.png
  </files>
  <read_first>
    - app/icon.png — must exist from Task 1 before running these commands
    - .planning/phases/04-launch-prep/04-RESEARCH.md — Pattern 6 (ImageMagick commands) and Pattern 7 (og.png spec: 1200×630, #0a0a0a bg, wordmark, tagline)
    - .planning/phases/04-launch-prep/04-UI-SPEC.md — OG image visual spec (logo mark 80px tall, centered at ~35% from top, "SDT tech" 36px semibold white 12px below mark, tagline 20px regular #a1a1aa 8px below wordmark, 60px safe zone)
  </read_first>
  <action>
    Run the following three ImageMagick commands in order:

    1. apple-icon.png (180×180 with 20px padding — logo at 140×140 centered):
    convert app/icon.png -resize 140x140 -gravity center -background "#0a0a0a" -extent 180x180 app/apple-icon.png

    2. favicon.ico (multi-size 32px + 16px):
    convert app/icon.png -resize 32x32 /tmp/icon32.png
    convert app/icon.png -resize 16x16 /tmp/icon16.png
    convert /tmp/icon32.png /tmp/icon16.png app/favicon.ico

    3. og.png (1200×630 with logo mark, wordmark, tagline):
    The logo mark in the OG image is the SVG geometry at 80px tall, centered horizontally, positioned so its vertical center is at ~35% from top (y≈220). Build the composite using ImageMagick:

    convert -size 1200x630 xc:"#0a0a0a" \
      /tmp/logomark.svg -resize x80 -gravity None -geometry +560+180 -composite \
      -fill "#ffffff" -font Helvetica-Bold -pointsize 36 \
      -gravity None -annotate +560+285 "SDT tech" \
      -fill "#a1a1aa" -font Helvetica -pointsize 20 \
      -gravity None -annotate +560+313 "Infrastructure for intelligent operations." \
      public/og.png

    If the -annotate geometry doesn't center cleanly, fall back to -gravity Center with adjusted offsets:
    convert -size 1200x630 xc:"#0a0a0a" \
      /tmp/logomark.svg -resize x80 -gravity Center -geometry +0-80 -composite \
      -fill "#ffffff" -font Helvetica-Bold -pointsize 36 \
      -gravity Center -annotate +0+20 "SDT tech" \
      -fill "#a1a1aa" -font Helvetica -pointsize 20 \
      -gravity Center -annotate +0+56 "Infrastructure for intelligent operations." \
      public/og.png

    Helvetica is acceptable as a fallback for Inter in this static OG image (per RESEARCH.md A3).

    Place apple-icon.png in app/ (NOT public/) — Next.js App Router auto-emits apple-touch-icon link from app/apple-icon.png.
    Place favicon.ico in app/ replacing the existing 16×16 stub.
  </action>
  <verify>
    <automated>

      identify app/apple-icon.png | grep -i "180x180" &&
      file app/favicon.ico | grep -i "MS Windows icon" &&
      identify app/favicon.ico | grep -i "32x32" &&
      file public/og.png | grep -i "PNG" &&
      identify public/og.png | grep -i "1200x630"
    </automated>
  </verify>
  <done>

    public/og.png exists as 1200×630 PNG.
    npm run build exits 0 (no TypeScript errors introduced — this task touches only binary files).
  </done>
