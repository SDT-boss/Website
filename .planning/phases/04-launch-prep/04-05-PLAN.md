<task type="checkpoint:human-action" gate="blocking">
  <what-built>
    The site is verified working at *.vercel.app. This step connects the sdt.technology custom domain. This requires action in two external dashboards (Vercel and Cloudflare) that cannot be automated via CLI.
  </what-built>
  <how-to-verify>
    Step A — Add domain in Vercel dashboard:
    1. Go to vercel.com/dashboard → select the sdt-technology project
    2. Settings → Domains → Add Domain → enter: sdt.technology
    3. Also add: www.sdt.technology (Vercel will set up www → apex redirect)
    4. Vercel will show DNS records to configure. Note the values (typically: A record 76.76.21.21 and CNAME for www)

    Step B — Configure Cloudflare DNS (CRITICAL: grey-cloud only):
    1. Log in to Cloudflare dashboard → select sdt.technology zone
    2. DNS → Records
    3. Add A record: Name "@", IPv4 "76.76.21.21", Proxy status: DNS Only (grey cloud icon — NOT orange cloud)
    4. Add CNAME record: Name "www", Target "cname.vercel-dns.com", Proxy status: DNS Only (grey cloud icon)
    5. Delete any existing A or CNAME records for @ and www that point elsewhere
    6. WARNING: If the proxy status shows orange cloud (Proxied), click it to toggle to grey cloud (DNS Only). Orange-cloud causes ERR_TOO_MANY_REDIRECTS with Vercel SSL.

    Step C — Verify domain in Vercel:
    1. Return to Vercel → Settings → Domains
    2. Wait for DNS propagation (2-10 minutes typical with Cloudflare)
    3. Vercel will show a green checkmark when DNS is verified and SSL certificate is issued
    4. Test: open https://sdt.technology in a browser — confirm HTTPS loads without certificate errors or redirect loops

    Note: Public traffic is now live on sdt.technology. Only proceed with Step A–C after owner has explicitly approved going live.
  </how-to-verify>
  <resume-signal>Type "domain-live" once sdt.technology resolves with HTTPS and green Vercel domain status, or "skip-domain" to defer domain connection to a later session</resume-signal>

|----------|-------------|

| browser → sdt.technology | HTTPS enforced by Vercel; no user auth or data collection on this static site |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-10 | Spoofing | Domain ownership | accept | Domain is owner-controlled; Vercel verifies ownership via DNS |
| T-04-11 | Denial of Service | Public site exposure | accept | Static HTML/CSS/PNG — no server-side execution surface; Vercel CDN handles load |
