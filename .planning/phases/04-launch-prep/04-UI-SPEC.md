## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, gap between 404 code and message |
| md | 16px | Default element spacing, button padding horizontal |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks (mobile section padding) |
| 3xl | 64px | Page-level spacing (desktop section padding: 96px exception) |

Exceptions:
- Section vertical padding: 96px desktop / 48px mobile (from DESIGN.md — already used in all prior phases)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` (`background`) | Page background, OG image background |
| Secondary (30%) | `#111111` / `#1a1a1a` (`surface` / `surface-2`) | Cards, navbar, footer surface |
| Accent (10%) | `#008684` (`cyber-jade`) | See reserved list below |
| Destructive | n/a | No destructive actions in this phase |

Accent (`cyber-jade`) reserved for:
1. The "Return to SDT tech" link on the 404 page
2. Hover state on 404 return link (`jade-strong` `#007069`)
3. Focus rings on all interactive elements

Accent (`volt-green`) is NOT used in Phase 4. The 404 page has no primary CTA — it uses the cyber-jade ghost link per the one-Volt-Green-per-page rule. The og.png static image may show the logo mark (which contains volt-green as the top leaf) but no CTA button appears in the image.

Status / semantic colors:
- `#52525b` (`text-muted`) — 404 code numeric string
- `#a1a1aa` (`text-secondary`) — 404 body message text
- `#2a2a2a` (`border-subtle`) — borders on the branded 404 page container (if bordered)

Source: CLAUDE.md "Volt Green rule", DESIGN.md "Color System", existing `not-found.tsx`

### LAUNCH-02: OG Image

| Asset | Path | Spec |
|-------|------|------|
| `og.png` | `/public/og.png` | 1200×630 PNG |

OG image visual spec:
- Background: `#0a0a0a` (solid fill — no dot-grid, no texture)
- Logo mark: centered horizontally, vertically positioned at ~35% from top, 80px tall (two overlapping leaf SVG shapes)
- Wordmark below mark: "SDT tech" in Inter semibold, 36px, color `#ffffff`, 12px below mark
- Tagline below wordmark: "Infrastructure for intelligent operations." in Inter regular, 20px, color `#a1a1aa`, 8px below wordmark
- No borders, no box shadows, no decorative elements
- Safe zone: 60px margin all edges (content stays within 1080×510 safe area)

This is a static PNG created manually (or via script). It does NOT use `next/og` — static export mode prevents runtime image generation.

Root layout OG metadata:
```

openGraph.images: [{ url: "/og.png", width: 1200, height: 630, alt: "SDT tech — Infrastructure for intelligent operations." }]
twitter.card: "summary_large_image"
twitter.images: ["/og.png"]
```

### LAUNCH-05: Branded 404 Page

The existing `app/[locale]/not-found.tsx` stub is functional but minimal. Phase 4 upgrades it to a branded page while keeping the same layout structure.

Visual spec:
- Layout: `flex flex-col items-center justify-center min-h-[60vh] gap-8 text-center px-6` (increase gap from 4 to 8 for breathing room)
- Logo mark: render `<LogoMark />` component at 40px height above the error block
- Error code: `"404"` — 14px, `font-mono`, `text-text-muted`, `tracking-widest`
- Heading: `"Page not found"` — 20px, weight 600, `text-white`, `leading-[1.2]`
- Body: `"The URL you followed doesn't exist or has moved."` — 16px, weight 400, `text-text-secondary`, `leading-[1.6]`
- CTA link: `"Back to home"` — 14px, `text-cyber-jade`, hover `text-jade-strong`, `transition-colors duration-150 ease-out`
- No Volt Green on this page (no primary CTA — this is an error state, not a conversion surface)
- No box shadow on any element
- `href` on the return link: `"/"` (root redirect handles locale routing)

## Implementation Notes for Executor

1. **No `next/og` or `@vercel/og`** — static export (`output: "export"`) prevents runtime image generation. `og.png` must be a pre-generated static PNG in `/public/`.
2. **`metadataBase` in root layout** — add to `app/layout.tsx`, not `app/[locale]/layout.tsx`, so it applies site-wide.
3. **Favicon ICO** — if creating from code, use `sharp` or an online tool. The `/public/favicon.ico` file must be present; Next.js App Router picks it up automatically.
4. **`not-found.tsx` scope** — `app/[locale]/not-found.tsx` catches 404s within locale routes. A root `app/not-found.tsx` may also be needed for routes that fall outside the `[locale]` segment (e.g., `/invalid-path`). Implement both stubs with the same visual spec.
5. **Vercel deploy (LAUNCH-06)** — no UI visual contract required; this is a devops task. DNS: Cloudflare grey-cloud (DNS only) records as noted in STATE.md blockers.
6. **Per-page metadata i18n** — implement English titles/descriptions only. Indonesian and Chinese metadata is a v2 enhancement; the static export will serve the English strings for all locales in Phase 4.
